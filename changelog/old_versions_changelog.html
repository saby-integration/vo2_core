Отправить УПД можно из раздела «Продажа/УПД» (СФ + Реализация). Если у документа реализации есть связанный счет-фактура, то выгружается УПД, который включает в себя фактуру и реализацию. Если связанного документа нет, выгружается УПД, который включает в себя только реализацию. Для подключения данного раздела укажите соответствующие настройки в разделе «Настройки/Файлы настроек».
  • Ускорена выгрузка документов.
  • Добавлен механизм сопоставления документов перед загрузкой. Для его включения в разделе «Настройки» установите флаг «Сопоставлять документы перед загрузкой». Если флаг установлен, перед загрузкой будет осуществляться поиск подходящего документа в 1С. Если будет найден один подходящий документ, то он автоматически сопоставится с загружаемым документом СБИС, иначе создастся новый документ.
  • Появилась возможность просмотреть несколько пакетов, сформированных по одному документу 1С (например, «Оказание услуг»).

Версия 2.0.24

  • Появилась отправка и загрузка универсального передаточного документа в формате, утвержденным приказом ФНС от 24 марта 2016 г. N ММВ-7-15/155@. По умолчанию выгружается документ, включающий в себя счет-фактуру и документ об отгрузке товаров (выполнении работ/оказании услуг). Отправить УПД можно из раздела «Продажа/УПД» (СФ + Реализация). Чтобы включить этот раздел, укажите соответствующую настройку в разделе «Настройки/Файлы настроек».
  • Реализована отправка прайс-листа в формате CommerceML из раздела обработки «Справочники/Номенклатура».
  • Добавлены функции внешнего интерфейса обработки. Они позволят отправлять документы, сопоставлять и утверждать полученные, обновлять статусы и обрабатывать служебные документы в автоматическом режиме без запуска внешней обработки. Функции можно вызывать из других внешних обработок или из модулей конфигурации (для измененных конфигураций).

Версия 2.0.23

В окне отправки укажите организацию-отправителя и контрагентов-получателей, напишите сообщение и/или приложите файлы с диска и нажмите кнопку «Отправить».
  • Неформализованные вложения (текстовые документы, картинки и др.) теперь можно открыть в другой программе прямо из окна просмотра пакета документов.
  • Появилась возможность отображать статусы документов не только во внешней обработке, но и в реестрах 1С в столбце «Состояние ЭДО». Для такой настройки в конфигурации 1С должен присутствовать регистр сведений «Состояния электронных документов».
  • Для конфигурации «Бухгалтерия 3.0» реализовано формирование документов (накладных и счетов-фактур) от обособленных подразделений. При таком способе формирования в документ подставляются реквизиты головной организации (кроме КПП).

Версия 2.0.22

  • Объединены обработки для веб-клиента и для «тонкого» клиента (управляемые формы). Для веб-клиента остается ограничение — подписание документов только ЭП с хранением на сервере.   • Реализована выгрузка акта приема-передачи прав для партнеров компании «Тензор».

Версия 2.0.21

  • Добавлены настройки для отправки накладных в ЕГАИС для конфигурации «Бухгалтерия предприятия», версии 2.0 и 3.0.
  • Указать адрес сервера online.sbis.ru теперь можно в файле сбисПараметры.xml в каталоге настроек. Пользователям Windows XP нужно прописать адрес в следующем виде: <АдресСервера>https://xp-online.sbis.ru/<АдресСервера>.
  • Появился механизм загрузки корректировочных документов. Корректно смогут загрузиться только документы, отправленные из новой (21-ой и выше) версии обработки. При выгрузке корректировки теперь указываются реквизиты исходного документа. Это необходимо для правильной загрузки документа у получателя.
  • Изменился механизм создания карточек номенклатуры для случаев, когда при загрузке документов не вся номенклатура сопоставлена. В новой карточке номенклатуры теперь возможно заполнить любые реквизиты значениями, которые определены по умолчанию. Для этого требуется дополнительная настройка.
  • Реализована поддержка новых форматов накладной и акта в соответствии с приказами ФНС РФ от 30 ноября 2015 г. № ММВ-7-10/551 и № ММВ-7-10/552. По умолчанию выгрузка документов происходит пока в старом формате (он действует до января 2017 года). Загружать в 1С можно документы как старого, так и нового формата.
